⚙️ CONFIGの書き換え方法
⚠️ teacher.html・student.html・admin.html の3ファイルすべてで書き換えが必要です
各ファイルをメモ帳・VSCodeで開き、先頭付近の CONFIG コメントを探してください。

① supabase.com → ログイン → 「New Project」でプロジェクト作成

② Project Settings → API → Project URL と Anon Public Key をコピー

③ aistudio.google.com → 「Get API key」で Gemini API Key を取得（無料）

④ 下の「テーブル作成SQL」をコピーして Supabase の SQL Editor で実行

⑤ 各HTMLファイルの CONFIG セクションに入力して保存

🗄️ テーブル作成SQL（Supabase SQL Editorで実行）

このSQLをコピーして、Supabase管理画面の「SQL Editor」に貼り付けて実行してください。

-- ① 学校マスターテーブル create table if not exists schools ( id uuid default gen_random_uuid() primary key, name text not null, code text unique not null, teacher_password text not null, created_at timestamptz default now() ); -- ② 採点結果テーブル create table if not exists submissions ( id uuid default gen_random_uuid() primary key, school_id uuid references schools(id) on delete cascade, student_name text, student_id text, grade text, class_name text, seat_number text, translation_type text, topic text, answer text, feedback text, status text default 'pending', submitted_at timestamptz default now(), graded_at timestamptz ); -- ③ Row Level Security（誰でも読み書き可能に設定） alter table schools enable row level security; alter table submissions enable row level security; create policy "public_schools" on schools for all using (true) with check (true); create policy "public_submissions" on submissions for all using (true) with check (true);
📁 ファイル構成

📄 teacher.html：先生用（PDFアップロード・採点・結果確認）

📄 student.html：生徒用（氏名で採点結果を確認）

📄 admin.html：管理者用（学校登録・全データ管理）

※ 3ファイルとも単体のHTMLファイルです。サーバー不要でブラウザで開くだけで動作します。
